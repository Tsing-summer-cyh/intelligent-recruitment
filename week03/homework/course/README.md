## 💻 核心技术实现

### 1. 前后端联调与跨域部署闭环
- **开发阶段**：通过 Vite 的 `server.proxy` 将 `5173` 端口的 `/api` 请求无缝转发至 Koa 后端的 `3000` 端口，解决开发环境跨域问题。
- **生产部署**：执行 `npm run build` 后，在 Koa 后端引入 `koa-static` 托管 `client/dist` 目录。同时，为了解决 React Router History 模式下刷新页面报 404 的问题，我在 Koa 中编写了路由兜底中间件，将所有非 `/api` 的请求全部打回到 `index.html` 交由前端路由接管。

### 2. 状态管理与路由鉴权机制
封装了统一的 `request.ts` (基于 Axios)：
- **请求拦截器**：自动从 LocalStorage 读取 Token 并注入到请求头 `Authorization: Bearer <token>` 中。
- **响应拦截器**：统一剥离了外层的 `code/msg/data`，并集中处理了 401 登录过期异常。
- 前端通过编写 `<PrivateRoute>` 高阶组件，在组件渲染前校验 Token，未登录或 Token 失效则自动重定向至登录页。

### 3. 复杂数据映射与文本溢出处理
在“学生管理”模块中，后端仅返回了每个学生选修课程的 `course_ids` 数组。
为了在表格中友好展示，我在 `render` 函数中将 `course_ids` 与提前请求好的 `courses` 列表进行比对，动态映射出课程名称。同时，考虑到学生选课较多时会撑爆表格，我巧妙使用了 Tailwind 的 `truncate max-w-[200px]` 工具类进行文本截断，并结合 `title` 属性实现了鼠标悬浮查看完整选课的优雅交互。

### 4. Markdown 动态渲染与排版引擎
在实现“学习总结”页面时，遇到 Tailwind CSS 会清空所有默认 HTML 标签样式的难题。
我没有选择手写繁琐的 CSS 重置代码，而是引入了官方的 `@tailwindcss/typography` 插件，仅使用一个 `prose` 类名就瞬间赋予了整个 Markdown 完美的排版效果。同时，为了让 MD 中的相对图片路径在路由中正常显示，我通过字符串 `.replace()` 正则替换，将 `assets/` 动态转为了后端的静态托管地址 `/api/static/assets/`。

## 💡 其他想对老师说的话

在这次从零开始的全栈实战中，我遇到了许多前端工程化中非常真实的问题，但也正是在解决这些问题的过程中，我的收获极大：
1. **TypeScript 类型校验**：深刻体会到了 TS 带来的严谨性。在联调初期，因为前后端数据格式不匹配报了不少红线，通过定义 `:any` 或补全接口类型，我学会了如何与严格的类型检查“和平共处”。
2. **工具链版本更迭**：在配置 Tailwind CSS 时，踩到了最新 V4 版本架构大改导致初始化报错的坑，最终通过指定 `tailwindcss@3` 版本完美解决，这让我明白了锁定依赖版本在工程化中的重要性。
3. **注重用户体验**：在基础需求之外，我主动优化了侧边栏固定不随页面滚动的体验，修复了 Dashboard 卡片高度不一致的问题，并细心清理了表单的默认选中状态，力求呈现一个真正具备“企业级体验”的后台管理系统。
