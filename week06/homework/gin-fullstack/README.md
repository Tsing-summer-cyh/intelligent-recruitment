## 核心技术实现

### 1. SQLite 环境迁移

- 将后端默认配置 `server/config.yaml` 的数据库类型改为 `sqlite`。
- 为 `sqlite` 设置默认数据库名 `gva` 和目录 `./data`，减少初始化时的手工配置。
- 在后端启动阶段自动创建 SQLite 数据目录，保证执行 `go run main.go` 时数据库文件可正常生成。
- 前端初始化页面默认选中 `sqlite`，并预填数据库目录，便于直接初始化系统。

### 2. 用户最后登录行为追踪

- 在 `sys_users` 模型新增 `last_login_ip` 与 `last_login_at` 字段。
- 登录成功后，在签发 token 前把当前请求 IP 和登录时间回写到用户表。
- 用户列表接口直接返回新增字段，前端无需额外发起请求。
- “用户管理”列表页新增“登录IP”“登录时间”两列，其中登录时间按 `YYYY-MM-DD HH:mm` 格式展示，空值显示 `-`。
