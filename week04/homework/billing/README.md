# 智能阶梯计费系统 (Intelligent Tiered Billing System)

---

## ✨ 已完成的功能
1. **基础阶梯电费计算：** 实现了精确到三档的用电量阶梯计费逻辑。
2. **峰谷时段动态调价：** 根据用户输入的用电时间，自动应用高峰期（+10%）和低谷期（-20%）的费率因子。
3. **系统自动初始化：** 利用 Go 语言的 `init()` 函数，在程序启动时自动打印计费规则版本号和系统当前时间。
4. **交互式命令行应用：** 通过终端引导用户输入用电量和用电时段，并打印格式化的精美账单明细。
5. **完整的单元测试：** 利用结构体和切片实现了表驱动测试，全面覆盖了各阶梯与峰谷时段的组合情况。

---

## 💡 关键逻辑实现思路
本项目的核心思路是**“逻辑拆分，单一职责”**：
1. **常量管理：** 将所有的用电阈值（200度、400度）、单价以及峰谷费率因子全部提取为 `const` 常量，避免代码中出现“魔法数字”，提高代码可维护性。
2. **分步计算：** 将复杂的计费过程拆分为两步。首先，`CalculateBaseFee` 专注于处理纯粹的阶梯数学计算；其次，`CalculateFinalFee` 在基础费用的之上，叠加峰谷时间因子的计算。
3. **时间解析：** 利用 `strings.Split` 将用户输入的 "HH:mm" 格式按冒号拆分，提取小时数转换为整型，从而判断是否落在 8:00-22:00 的高峰区间内。

## 🛠️ 函数列表及其功能说明

| 函数名 | 功能说明 |
| :--- | :--- |
| `init()` | 系统初始化函数。在 `main` 之前自动执行，用于打印系统当前时间和版本号。 |
| `CalculateBaseFee(usage float64)` | 基础阶梯电费计算函数。接收总用电量，根据三档规则计算并返回基础费用。 |
| `IsPeakTime(timeStr string)` | 时段判断函数。解析 "HH:mm" 格式的字符串，判断是否属于高峰时段，返回布尔值。 |
| `CalculateFinalFee(usage float64, timeStr string)` | 最终电费计算函数。整合基础电费和峰谷时段因子，输出最终需缴纳的金额。 |
| `RunBillingSystem()` | 交互执行引擎。负责处理终端的控制台 I/O（引导输入、接收参数、格式化打印账单）。 |
| `main()` | 程序的主入口，内部调用 `RunBillingSystem()` 启动系统。 |

## 🧗 处理“跨阶梯计费”时遇到的逻辑难点
在实现 `CalculateBaseFee` 时，最大的逻辑难点在于**如何正确累加跨多个阶梯的费用，而不发生重复计算或遗漏。**

**难点描述：**
一开始很容易陷入误区，比如当用电量为 300 度（属于第二档）时，直接用 `300 * 第二档单价`，这显然是不符合现实中阶梯电价“超额累进”规则的。正确的做法是：前 200 度必须按第一档价格算，**只有超出 200 度的那 100 度**才能按第二档价格算。

**解决过程：**
我通过 `if-else if-else` 结构结合简单的数学拆分解决了这个问题：
* **第一档：** 直接 `usage * Tier1Price`。
* **第二档：** 将费用拆分成两块：`满额的第一档费用 (200 * 0.5)` + `超出第一档的部分 ((usage - 200) * 0.8)`。
* **第三档：** 将费用拆分成三块：`满额的第一档费用` + `满额的第二档费用` + `超出第二档的部分 ((usage - 400) * 1.2)`。
这种分块累加的方式使得代码逻辑非常严谨，且完全贴合了现实生活中的计费方式。
