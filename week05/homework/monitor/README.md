## 完成的功能
1. **配置解析**：能够读取外部的 `config.json`，自动识别 HTTP 和 TCP 不同的探测协议。
2. **并发探测**：用 Goroutine 实现了多目标的并发扫描。为了验证并发效果，在代码里人为加了 `time.Sleep`，运行发现所有任务确实是同时发起的。
3. **超时控制**：用 `context.WithTimeout` 处理了超时问题，遇到像 httpbin 这种会卡住的服务，时间到了会自动掐断，不会导致整个程序死等。
4. **并发数据收集**：结合 `Channel` 和 `sync.WaitGroup`，安全地把各个协程的探测结果汇总到了一起，最后才触发日志生成。
5. **重试机制**：实现了失败重试，根据配置里的 `retry_count` 字段决定重试次数。
6. **日志报表**：跑完之后会生成类似 `monitor-log-2026xxxx.log` 的文件，里面统计了成功/失败的数量，还能找出响应最慢的服务。

## 开发中遇到的问题与解决
**遇到问题：并发收集数据时发生冲突（Data Race）**
最开始做结果聚合的时候，我直接让所有的 goroutine 往同一个 slice 里面 `append` 结果。测试的时候发现有时候最终输出的条数对不上，甚至直接报错。查资料才知道，多个协程同时去修改同一个变量会发生数据竞态。

**怎么解决的：**
我没有用 `sync.Mutex` 加锁，因为感觉会影响并发性能。后来改用了 Go 推荐的 Channel 来做。
我建了一个缓冲 Channel，各个并发任务拿到结果后，只管往 Channel 里发送数据。主协程这边用 `WaitGroup` 等所有任务执行完，然后关闭 Channel，最后再用一个循环把 Channel 里的数据统筹提取出来生成日志。这种做法不用加锁，代码写起来也更优雅，彻底解决了并发写数据的问题。
