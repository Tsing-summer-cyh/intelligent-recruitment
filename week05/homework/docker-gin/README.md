## 三、 项目简介与架构说明
本项目是一个基于前后端分离架构的“AI 辅助英语学习 Web 应用”。
用户可以通过前端页面查询生词，后端系统会调用大语言模型（LLM）生成该单词的精准释义和多条例句。查询结果返回前端展示后，用户可将其持久化保存到个人的云端单词本中，并支持分页浏览与管理。

**技术栈架构：**
* **前端**: Vue 3 + Vite + Vue Router + Axios
* **后端**: Go 1.21 + Gin + GORM + JWT
* **数据库**: MySQL 8.0
* **AI 模型**: DeepSeek (或通义千问)
* **部署**: Docker + Docker Compose + Nginx

## 四、 运行指南 (核心考察点)

### 1. 前置依赖
在运行本项目前，请确保您的服务器或本地计算机已安装以下软件：
* [Docker](https://www.docker.com/)
* [Docker Compose](https://docs.docker.com/compose/)

### 2. 配置 AI 的 API Key
项目依赖外部大语言模型提供翻译服务。请在项目的 `backend` 目录下创建一个名为 `.env` 的环境变量文件，并填入您的真实 API 密钥：

```env
AI_API_KEY=你的真实DeepSeek或通义千问密钥
```

### 3. 一键启动命令
本项目的数据库建表（`docs/init.sql`）以及前后端网络通信已完全通过 Docker Compose 编排。无需人工干预数据库，直接在**项目根目录**（即 `docker-compose.yml` 所在目录）执行以下命令：

```bash
docker-compose up -d
```
*(注：如果代码有更新，请加上 `--build` 参数重新构建镜像)*

### 4. 访问服务
为了避免与宿主机默认端口冲突，本项目前端映射至 8088 端口，数据库映射至 3307 端口。服务启动完成后：
* **访问前端页面**：在浏览器中打开 `http://localhost:8088`
* **后端 API 地址**：Nginx 已将前端所有对 `/api/*` 的请求反向代理至后端的 `8080` 端口。例如请求 `http://localhost:8088/api/login` 实际访问的是 Go 后端的 `/login`。

### 5. 停止服务
如需停止并移除容器，请在根目录执行：
```bash
docker-compose down
```
